Inventory Tracking System

The Inventory Application “Diva Reports”, provided the ability for company employees to pull inventory information from the database according to different options such as vendors, date ranges, statuses, product categories, the amount of sales etc. The Inventory Tracking application consists of multiple tabs. All the tabs are listed below with appropriate descriptions and screen shots (please click on the name of the tab to view its screen shot in a separate widow). All the code for the application can be viewed on GITHUB and the SQL Scripts for database tables can be retrieved from THIS LINK on github, as well.

  • The tab “Sales of items” provides information about all items available at the company warehouse. The inventory can be selected by Accounts (vendors) on the left side and product names on the right side of the form. The date ranges and various check mark options allow to display information in a preferable format. The buttons “Get Items” - retrieves all items, “Get Item Sales” - displays the amount of sales for every single item and the button “Search” - allows to find any specific item within the generated results. The button “Export To CSV” - provide capability to export the results in the CSV format for further reviews. All other tabs have similar functionality and they are self-explanatory from the provided screen shots.
  • The tab “Current Sold” provides inventory information about the items currently sold.
  • The tab “Par Percentage” shows percentage of sales based on the formula InStock Inventory / (New Par + Par Correction)
  • The tab “Out of Stock” provides information about the items that are out of stock based on the prior tab and multiple criteria such as “New Par”; “Par Correction”; “Par Percentage”; “On Order”; “Discontinued”;
  • The tab “Bulk Export” provides various information about products that are sold on export.
  • The tab “Sales of Sets” contains information about products that can be sold only in sets.

Inventory Scanning Application verifies if an item exists in the company warehouse based on bar codes and items from Excel file. The functionality of the application can be described in the following steps: 1) when the Inventory Scanning Application is executed it connects to Excel file with the list of specific items that are available at the company warehouse; 2) A user scans an item and the bar code is displayed in a separate text field in the application; 3) The user clicks the button “search” and the application searches the items based on the bar code in excel sheet. If the item exists in the excel file then the item is added automatically to the list in the Inventory Scanning Application. If the item doesn’t exist in the list then the application makes the sound and displays the dialog box for the user to enter item number and quantity. 4) All information, collected in the list in the Inventory Tracking application, can be exported into separated CSV file for further review and processing. The excel file was used instead of MS SQL Server database for this particular application only. The source code of the application can be viewed on this link on GITHUB.
